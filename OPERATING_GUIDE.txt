# FX Command Agents v12 — OPERATING GUIDE
# ════════════════════════════════════════════════════════════
# Three phases: PRE-FLIGHT → SIM TESTING → LIVE
# ════════════════════════════════════════════════════════════

# ─────────────────────────────────────────────────────────────
# PHASE 1 — PRE-FLIGHT (do this ONCE, takes ~10 minutes)
# ─────────────────────────────────────────────────────────────

STEP 1: Copy files to working folder
─────────────────────────────────────
   Unzip fx_agents_v12.zip to:
   C:\fx_agents\

   (Don't run from Downloads — path spaces cause MT5 issues)

STEP 2: Install Python dependencies
─────────────────────────────────────
   Open cmd.exe as Administrator, run:

   pip install MetaTrader5 pandas numpy requests feedparser anthropic

   Verify:
   python -c "import MetaTrader5, pandas, anthropic; print('OK')"

STEP 3: Configure MT5 desktop
─────────────────────────────────────
   • Open MetaTrader5
   • Log in to your demo account (107377015)
   • Tools → Options → Expert Advisors → tick:
       ✅ Allow automated trading
       ✅ Allow DLL imports
   • Make sure all 12 symbols are visible in Market Watch:
       AUDUSD EURUSD EURJPY GBPUSD GBPJPY
       NZDUSD USDCAD USDJPY XAUUSD
       USDCHF EURGBP CADJPY          (v13 promoted — required)

STEP 4: Add XAUUSD M15 data to MT5
─────────────────────────────────────
   In MT5: File → Open Data Folder → find history folder
   Or easier: open an XAUUSD M15 chart, scroll back as far
   as it will go — this forces MT5 to cache the history.

STEP 5: Run power setup (once)
─────────────────────────────────────
   Right-click setup_windows.bat → Run as administrator
   This stops Windows sleeping and cutting the connection.

STEP 6: Run the diagnostic
─────────────────────────────────────
   cd C:\fx_agents
   python test_full_system.py

   Every line should say [PASS]. If anything says [FAIL]:
   • MT5 not running → open MT5 first
   • Symbol not found → add to Market Watch
   • Telegram [FAIL] → check bot token in config.py

─────────────────────────────────────────────────────────────
# PHASE 2 — SIM TESTING (current mode, ~2-4 weeks)
# config.py has SIM_MODE = True → zero real money risk
# System fires real signals, paper-executes them, logs everything
─────────────────────────────────────────────────────────────

STEP 7: Start the system
─────────────────────────────────────
   OPTION A — double-click start_agents.bat (easiest)
   OPTION B — cmd:
       cd C:\fx_agents
       python main_agents.py

   You will see:
   [KIRA] Scanning EURUSD...
   [KIRA] Scanning XAUUSD...
   [GUARD] Balance: $3000 | DD: 0.0%
   etc.

   The system scans every symbol every 4 minutes.
   When a signal fires, your Telegram receives:
   ━━━━━━━━━━━━━━━━━━━━━
   🟡 SIM SIGNAL — Grade A
   BUY EURJPY | CBE | COMPRESSING
   Entry: 162.450 | SL: 162.180 | TP: 163.540
   R:R 1:3.9 | Risk: 1%
   KIRA 82 | NOVA ✅ | ATLAS 71 | GUARD 90
   ━━━━━━━━━━━━━━━━━━━━━

WHAT TO WATCH IN SIM:
─────────────────────────────────────
   After every 10 signals, check:
   • SIM WR tracking toward 49.5%? (±10% is fine at small samples)
   • Any NOVA news blocks firing correctly?
   • Any GUARD exposure blocks?
   • Telegram arriving < 5 seconds of signal?

   After 30 signals, run:
   python -c "
   from agent_atlas import AgentATLAS
   a = AgentATLAS()
   print(a.get_stats())
   print(a.get_sl_tp_advisory())
   "

   And run ORACLE attribution:
   python -c "
   from agent_oracle import AgentORACLE
   o = AgentORACLE()
   import json; print(json.dumps(o.attribution_report(), indent=2))
   "

STEP 8: First SAGE weekly review
─────────────────────────────────────
   After 7 days of SIM:
   python agent_sage.py

   Claude will read your journal and deliver a weekly review
   to Telegram. This is your first real AI agent loop working.

PASS CRITERIA for moving to live:
─────────────────────────────────────
   ✅ 30+ SIM signals fired without crashes
   ✅ SIM WR within 10pp of 49.5% (so 39-59%)
   ✅ SIM EV per signal > RM8 (half the backtest)
   ✅ No GUARD limit breaches
   ✅ Telegram delivery working
   ✅ SAGE review makes sense

─────────────────────────────────────────────────────────────
# PHASE 3 — LIVE (after SIM criteria met)
# Takes ONE line change in config.py
─────────────────────────────────────────────────────────────

STEP 9: Switch to live
─────────────────────────────────────
   In config.py, change ONE line:
       SIM_MODE = False

   First live signals will be marked 🟢 LIVE (not 🟡 SIM).
   GUARD enforces 1% risk — on $3,000 = ~$30 at risk per trade.
   Your GUARD capital-adequacy rule auto-applies 0.7× risk
   until balance reaches $4,500, then normalises to 1.0×.

STEP 10 (later): Prop Firm Mode
─────────────────────────────────────
   After 3 profitable live months:
   1. Pass a prop challenge (FTMO / MyForexFunds / The5ers)
      — with PROP_MODE=True for daily -3% / total -7% hard stops
   2. In config.py:
       SIM_MODE    = False
       PROP_MODE   = True
   3. Same edge on $100,000 funded = ~$1,700+/month

─────────────────────────────────────────────────────────────
# TROUBLESHOOTING QUICK REFERENCE
─────────────────────────────────────────────────────────────

"No signals firing for 3+ days"
   → Normal. System fires 6/month = ~1.5/week.
   → Check: python test_full_system.py

"MT5 connection lost"
   → start_agents.bat auto-restarts python main_agents.py
   → MT5 itself must stay running — check it hasn't logged out

"GUARD STOPPED appearing"
   → DD hit 15% threshold — system protecting capital
   → Check trade_history.json, review with SAGE

"Signal fired but Telegram late / missing"
   → Check Telegram bot permissions
   → python -c "from telegram_commands import *; test_telegram()"

"NOVA blocking too many signals"
   → Normal on high-impact news weeks (FOMC, NFP, BOJ)
   → Check: upcoming events in FF calendar
